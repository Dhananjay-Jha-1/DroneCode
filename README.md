This repository contains Python scripts for drone mission planning, simulation, and control using PyMavlink. The primary focus is on generating survey flight paths from KML files and executing them in an ArduPilot SITL (Software-In-The-Loop) environment. The project demonstrates a simulated search-and-rescue scenario involving multiple drones.

## Features

*   **KML Polygon Parsing**: Extracts mission boundaries from KML files.
*   **Lawnmower Path Generation**: Creates efficient, overlapping survey paths (lawnmower pattern) covering a defined polygonal area.
*   **Coordinate Projection**: Converts GPS coordinates (WGS84) to UTM for accurate metric calculations and back.
*   **Mission Control with PyMavlink**: Connects to drone simulations, uploads waypoints, and controls mission execution (arming, setting mode, starting mission).
*   **Multi-Drone Simulation**: Includes a ground station script to coordinate a "scan" drone and a "delivery" drone for a search-and-rescue simulation.

## File Descriptions

*   `lawnMower.py`: Parses a KML file, generates lawnmower-style waypoints based on specified altitude and camera FOV, and saves the resulting GPS coordinates to `waypoints.csv`.
*   `scanFinal.py`: A complete, self-contained script that generates waypoints from a KML file and immediately uploads them to a running SITL instance to start a survey mission.
*   `uploadScanWaypoints.py`: Reads pre-generated waypoints from `waypoints.csv` and uploads them as a mission to a connected drone simulation.
*   `ground_station.py`: Simulates a ground control station that manages two drones. It commands a scan drone to fly a mission, "detects" a survivor at a hardcoded location, and then sends a delivery drone to that location.
*   `parseKML.py`: A basic utility script to parse a KML file and extract polygon coordinates.
*   `step2.py`: A developmental script demonstrating the projection of KML polygon coordinates into a UTM coordinate system and calculating camera swath width.
*   `waypoints.csv`: An example output file containing latitude and longitude waypoints generated by `lawnMower.py`.

## Getting Started

### Prerequisites

*   Python 3
*   ArduPilot SITL
*   Install the required Python libraries:
    ```bash
    pip install pymavlink shapely pyproj
    ```

### Setup and Usage

#### 1. ArduPilot SITL Setup

You must have ArduPilot's SITL running before executing the mission scripts.

**For single-drone simulations (`scanFinal.py`, `uploadScanWaypoints.py`):**

Open a terminal and start the SITL instance. This will default to using UDP port `14550`.

```bash
sim_vehicle.py -v ArduCopter --map --console
```

**For the two-drone simulation (`ground_station.py`):**

You need to launch two separate SITL instances on different ports.

In the first terminal (Scan Drone):
```bash
sim_vehicle.py -v ArduCopter --map --console -I0 --out=udp:127.0.0.1:14550
```

In a second terminal (Delivery Drone):
```bash
sim_vehicle.py -v ArduCopter --map --console -I1 --out=udp:127.0.0.1:14560
```

#### 2. Running a Scan Mission

Before running, ensure the `kml_file` path in the scripts points to your KML file.

**Option A: Generate waypoints and fly in one step**

This script handles everything from KML parsing to mission execution.

```bash
python scanFinal.py
```

**Option B: Generate waypoints first, then fly**

First, generate the waypoints from your KML file. This will create `waypoints.csv`.
```bash
python lawnMower.py
```
Then, upload the mission from the generated CSV file.
```bash
python uploadScanWaypoints.py
```

#### 3. Running the Two-Drone Search & Rescue Simulation

After launching the two SITL instances as described above, run the ground station script.

```bash
python ground_station.py
```

The script will connect to both drones, command the scan drone to start its mission, wait for it to reach the simulated survivor location, and then dispatch the delivery drone.
